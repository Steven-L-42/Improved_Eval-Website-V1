Verhalten des Evaluators:

  • Konzentrieren Sie sich ausschließlich auf die Durchführung der Bewertung ohne Rücksicht auf Fristen.
  • Bewahren Sie Professionalität während der Diskussion und halten Sie sich an die Bewertungskriterien.
  • Nach der Bewertung sollten Diskussionen abgeschlossen sein oder, falls fortgeführt, auf konstruktiver und objektiver Ebene gehalten werden.
  • Normalisieren Sie die Suche nach einer Zweitmeinung, insbesondere bei der Bewertung unbekannter Projekte (42 Heilbronn ist eine Peer-Learning-Universität).
  • Streben Sie einen hohen Qualitätsstandard bezüglich der Code-Struktur und Lesbarkeit an.
  • Teilen Sie Ihr Wissen über Code und vermeiden Sie das Durchfallen wegen kleiner Fehler wie Tippfehler (Denken Sie darüber nach, was Menschen von Ihnen lernen können, wenn Sie sie Fehler machen lassen).
  • Priorisieren Sie faire Bewertungen und vermeiden Sie manipulierte Bewertungen, um enge Bekannte zu begünstigen (Ihre Freunde).
  • Wenn Sie einen Tester verwenden, der Fehler anzeigt, müssen Sie in der Lage sein zu erklären, was der Tester tut und warum der Code fehlschlägt. Andernfalls ist das kein Grund für ein Nichtbestehen.

Verhalten des Bewerteten:

  • Übernehmen Sie die Verantwortung für Ihr Projekt, lernen Sie aus Problemen bei der Einreichung und erkennen Sie Misserfolge als Lernmöglichkeit an.
  • Vermeiden Sie es, Bewertungen mit voreingenommenen Meinungen zu manipulieren, um faire Lernmöglichkeiten sicherzustellen (Buchen Sie keine Bewertungen mit Ihren Freunden).
  • Geben Sie den Evaluatoren ehrliches Feedback darüber, wie sie die Bewertung durchgeführt haben.
  • Buchen Sie nur eine Bewertung für einen einzelnen Slot und lassen Sie genügend Raum für die Diskussionen (Eine halbe Stunde kann ziemlich kurz für eine gründliche Bewertung sein).
